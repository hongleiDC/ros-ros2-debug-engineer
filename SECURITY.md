# Security Policy

## Supported version

安全修复仅面向 `main` 分支的最新版本。历史提交不会单独维护安全补丁。

## Reporting a vulnerability

请不要在公开 Issue 中披露可利用漏洞、访问令牌、DDS Security 材料或生产机器人信息。优先使用 GitHub 的私密漏洞报告功能；如果该功能不可用，请先联系仓库所有者，再安全地提供最小复现。

报告应包含：

- 受影响文件和版本；
- 影响范围与前置条件；
- 最小复现或概念验证；
- 建议修复或缓解措施；
- 是否涉及真实硬件、网络、凭据或敏感数据。

## Scope

重点关注：

- 路径遍历、任意文件覆盖或不安全删除；
- 命令注入和不受控子进程；
- 通过日志、快照或参数 dump 泄露秘密；
- bag 回放、service/action 或硬件激活越权；
- 知识记录并发损坏或验证绕过；
- GitHub Actions 供应链风险。

普通 ROS 项目的代码错误、设备配置问题和不包含安全影响的诊断误差，请使用常规 Issue。
