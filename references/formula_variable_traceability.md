## 核心原则

任何涉及数学模型、坐标变换、滤波、标定、时间同步、优化、误差传播、控制、统计指标或阈值计算的诊断与代码修改，都必须保留完整推导链。不得只给最终公式、最终数值或经验参数。

每一步必须能够回答：

1. 已知条件是什么；
2. 使用了哪一个公式或定义；
3. 公式中每个符号对应代码中的哪个变量、参数、消息字段或日志字段；
4. 单位、坐标系、时间基准、方向和索引约定是什么；
5. 从上一行到下一行做了什么代数、几何或统计变换；
6. 得出的中间量和结论如何服务于当前 `SC-*`。

## 强制推导格式

对关键结论使用下面的结构，不得跳步：

```text
目标判据：SC-*
已知：变量、来源、数值、单位、frame、时间基准
定义：符号及物理含义
公式：原始公式
变量映射：数学符号 <-> 代码变量/配置参数/消息字段
推导：逐行变换，每一行说明依据
代入：代入实际数值并保留单位
结果：数值、单位、有效位数和不确定度
检查：量纲、符号、方向、边界条件和极端情况
结论：该结果支持、否定或无法判断哪个假设
```

如果某一步缺少证据，写 `unknown`，不得用默认值或经验值补齐。

## 变量映射表

每个关键公式都维护映射表：

| 数学符号 | 代码变量 | 配置/消息来源 | 物理意义 | 单位 | frame/方向 | 有效范围 |
|---|---|---|---|---|---|---|
| `\Delta t` | `imu_time_offset_s` | `config/time_sync.yaml` | IMU 到系统时钟偏移 | s | sensor -> host | project evidence |

要求：

- 一个代码变量只对应一个明确物理量；同名变量不得在不同作用域表示不同单位或方向。
- 同一数学符号在同一推导中含义必须唯一。
- 代码变量名优先由物理量和单位组成，例如 `angular_velocity_rad_s`、`time_offset_s`、`position_map_m`。
- 禁止无语义关键变量名，如 `a`、`b`、`tmp`、`val`、`data2`、`x1`，除非它们是局部、极短生命周期并且紧邻公式映射说明。
- 不得把弧度变量命名为 `*_deg`，不得把毫秒值存入 `*_s`，不得隐藏坐标系。
- TF、四元数和外参变量必须在名称或类型旁说明方向，例如 `T_map_base` 表示把 `base` 中坐标变换到 `map`，不得仅写 `transform`。
- 矩阵、向量和协方差必须说明维度、排列顺序和左右乘约定。

## 代码实现规则

实现公式时：

1. 在代码附近引用公式编号或推导记录 ID，例如 `EQ-03`、`DER-0004`。
2. 变量命名必须与映射表一致；若受现有 API 限制，添加别名映射说明。
3. 常数必须有来源、单位和适用范围，禁止裸魔法数。
4. 单位转换必须显式，例如 `milliseconds * 1e-3`，并测试转换边界。
5. 对关键中间量增加断言、范围检查或诊断日志。
6. 测试至少覆盖：单位一致性、符号方向、零值、边界值和一个可手算样例。
7. 优化器、滤波器或估计器的状态向量必须列出索引到物理量的映射。

## 公式或模型发生变化

修改模型、符号定义或变量语义时，必须同时更新：

- 推导说明；
- 变量映射表；
- 代码和配置参数名；
- 单元与回归测试；
- 实验记录中的公式版本或推导 ID；
- 受影响的历史结论。

不得只改代码实现而保留旧公式说明，也不得只改文档不更新实现。

## 审查门禁

以下任一情况存在时，不得宣称实现正确：

- 关键步骤没有推导；
- 公式符号没有对应到真实代码变量；
- 变量单位、frame、方向或时间基准不明确；
- 变量名与物理含义不一致；
- 使用魔法数但没有来源；
- 代数正确但量纲不一致；
- 数值结果没有边界或手算样例验证；
- 代码中的状态顺序与论文/公式顺序不一致且没有映射。

## 持久化到项目知识库

当项目已经采用 `.ros_debug_project.yaml`，或用户授权维护项目知识时，公式和变量关系不能只出现在聊天、注释或临时表格中。必须按 [推理、公式与代码逻辑知识库](reasoning_knowledge_base.md) 写入：

- `formulas/FORM-*.yaml`：公式版本、符号、假设、约定和推导；
- `variable_mappings/MAP-*.yaml`：公式符号与代码变量、参数、消息字段、日志字段的对应；
- `reasoning_chains/REAS-*.yaml`：每一步前提、规则、操作、证据、反例检查和结论；
- `audits/AUD-*.yaml`：修改前后的自动审计结果。

代码和知识记录必须指向同一 commit。变量、单位、frame、方向、时间基准、状态索引或公式版本改变时，旧映射必须标为 deprecated 或更新为新版本，不能继续作为验证依据。
